1 · 主题陈述
中心问题:我们在工程化感知时,究竟在工程化什么?
What do we engineer when we engineer perception?

综述对象 = perceptual engineering:有意微调 / 增强 / 替代 / 衰减人类感官知觉的交互技术。反身性的答案是:HCI 大量借用神经科学的词与工具,却没把医学的纪律与伦理一并搬过来。我们用一条临床用来理解神经多样性的真实通路(受器 → 门控 → 脑岛整合 → 情感;此为主通路,可能并存其他通路)当尺,问 HCI 在为谁、用什么、作用于哪一层、有多有据、合不合伦理,从 autism / neurodiverse 切入推普适,为有据、可问责、可及的 wearable AI 立议程。

2 · 核心纳入判据

技术必须「作用于知觉本身」,而非只是「呈现信息」。EMS 让手动、GVS 改变平衡、热致幻、电子味觉、感官替代 → 纳入;普通 AR 卡片、普通通知 → 排除。

3 · Scope 五维
维度推荐决策
感官域 全九感纳入:视 / 听 / 触压 / 痛温 / 本体 / 前庭 / 内感受 / 嗅 / 味。与 ADSR2 故意分道:ADSR2 因「难可穿戴测」排除嗅味,但感知工程 HCI 的嗅味子领域活跃,本综述纳入。
作用模态 电 EMS/GVS/电触觉/电味/tDCS·tACS机械触觉 振动/力反馈/超声/pseudo-haptic热 Peltier/热致幻化学 嗅显示/味觉光与声(错觉级,非内容级)感官替代 跨模态映射生理反馈 HRV/EDA 闭环
时间窗 2013–2025(双侧交汇)。医学侧:DSM-5(2013)统一为单一 ASD、改维度化严重程度分级、首次将「感觉异常」列为核心特征(2003 前研究多被推翻 → 自然排除)。HCI 侧:可穿戴 / 泛在计算与生物信息成为 daily device。Bach-y-Rita 1969 等奠基只在 Background 引,不进 corpus。
数据库 / venue ACM DL + IEEE Xplore + Scopus。venue:CHI / UIST / TEI / DIS / CHI EA / Augmented Humans / IMWUT·UbiComp / ISS / MobileHCI;IEEE:World Haptics / ToH / IEEE VR。Scopus 跨学科补全 + snowballing;PubMed 选配(RQ4 神经依据交叉核对)。
纳入 / 排除 纳入:有 HCI/交互设计贡献且有意作用于知觉。排除:纯信息展示 UI;仅作 evaluation / 评估手段者(技术须作为 intervention 干预,而非评估工具);纯临床疗效试验(无设计贡献);非 autism 条件的临床假体(人工耳蜗 / 视网膜假体 / TENS)→ 仅 Background/Future;纯 read-out/控制型 BCI;非英文;二次综述。
4 · 两位专家专属决策

给医生(临床 / 神经边界)

  • 神经刺激(tDCS / tACS / TMS)纳入还是单列?推荐:HCI venue 内、针对知觉的纳入;侵入式 BCI 另案。
  • 已定:临床搜索锚定 autism 的诊断与治疗,故人工耳蜗 / 视网膜假体 / TENS 镇痛不进搜索,仅进 Background + Future Work。(这些是为聋 / 盲 / 疼痛而非 autism,落在 autism dx/tx 搜索范围外。)
  • 背书 RQ4 编码口径:神经科学被怎么用(motivation / mechanism / evaluation / inspiration / 缺席 × 承重);验证靠 biosensor / 客观指标,还是只靠自评量表(注:自评对内感受 sensibility 等主观维度合理,配 HR / 呼吸 / EDA 更可靠;veneer 特指「神经机制 claim 仅以自评单独背书、无客观锚点」);是否核算对神经通路干预的后果与伦理(如 body schema 改写后的可逆性、幻肢风险)。

给 HCI researcher(领域边界)

  • VR/AR 切法:知觉级(redirected walking / 跨模态错觉 / 感官重映射)纳入、内容级排除?推荐:是。
  • 感官替代 / 辅助设备(vOICe / BrainPort)算 perceptual engineering?推荐:纳入,是 margins→everyday 的桥。
  • venue / 模态有无遗漏(SIGGRAPH 知觉工作?ToH 触觉错觉?)。
