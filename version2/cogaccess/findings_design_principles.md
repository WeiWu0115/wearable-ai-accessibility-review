# 认知无障碍设计原则：Scoping Review 综述报告

> 重点人群：**自闭症 (ASD)** · 对比覆盖：智力/学习障碍、痴呆/老年认知衰退、ADHD/阅读障碍、获得性脑损伤/失语
> 数据库：ACM DL + IEEE Xplore · 时间窗 2015–2025 · 中英混合
> 配套文件：`protocol.md`（方法）· `corpus.md`（文献库）· `figures/`（图表）

---

## 0. 摘要 (Executive Summary)

认知无障碍研究的核心张力是：**WCAG 等主流标准对认知障碍覆盖不足，合规 ≠ 认知可用**。这一判断在多个人群文献中反复出现 [B1, F3, F4]。围绕这一空白，HCI/无障碍社区为不同人群提炼出大量设计原则。本综述发现：

1. **存在一套跨人群共性的"认知无障碍核心原则"**，可用 W3C COGA 的八大目标 [F1] 作为统一骨架：*清晰语言、降低记忆依赖、降低认知负荷/专注支持、可预测一致、防错与反馈、帮助与支持、可定制个性化、多模态冗余表征*。
2. **自闭症 (ASD)** 在共性原则之上有鲜明的**人群特异性**：**感官调节 (sensory regulation)**、**可预测性/抗变化 (predictability)**、**字面化语言 (literal language)**、以及对**深度定制**的强需求 [A1, A5, A8, A11]。
3. **方法学正在从"专家共识指南"转向"参与式/共同设计"**，尤其是成人 ASD 与 IDD 人群 [A10, A13, A14, B3, B4]。
4. **证据空白**：原则多为专家提炼或小样本，缺乏跨人群、跨情境的对照验证；自闭症成人、职场、AI 时代界面是新兴增长点 [A12, D5]。

## 1. 重点：自闭症 (ASD) 的设计原则 — RQ1

自闭症最系统的设计原则锚点是 **GAIA**（Britto & Pizzolato）[A1]：从 2005–2015 的 17 项研究中归纳 107 条推荐，经亲和图归并为 **28 条、10 个模块**。结合 ACM/IEEE 近十年文献，自闭症设计原则可归为七大主题：

### 1.1 感官调节 Sensory regulation（ASD 最突出特异性）
- 避免/可关闭动画、闪烁、自动播放；提供降低感官刺激的选项 [A5, A11]。
- 警惕"感官过载"——是自闭症成人压力的核心触发源 [A11]。
- 谨慎使用高饱和色彩与非常规字体（被视为干扰源）[A1-VVT/MT, GAIA]。
> GAIA 模块对应：**MT 多媒体**、**EGM 参与/专注**、**VVT 视觉文本词汇**。

### 1.2 可预测性与抗变化 Predictability / consistency
- 界面元素应"无需深度探索即可识别其功能"（**RP 识别与可预测性**）[A1]。
- **界面频繁/微小变化会给自闭症用户带来不适**，降低可用性与舒适度——应保持稳定、提供变化预告 [A8]。
- 一致的视觉与交互模式、清晰的步骤 [A1-RP/NVG]。

### 1.3 清晰、字面化的语言 Clear & literal language
- 文本可达性：减少隐喻、反讽、模糊；使用字面、直接的语言 [A4, A1-VVT]。
- 图文配合、redundant representations（**RR 冗余表征**：信息不绑定单一格式）[A1-RR]。

### 1.4 反馈与系统状态 Feedback & visibility
- **缺失或不完整的反馈对自闭症用户尤为关键**，尤其针对错误操作 [GAIA-VES/RA]。
- 明确的任务进度、状态可见性、对动作的即时回应 [A1-VES/RA]。

### 1.5 深度可定制 Customization（关键缺口）
- 字体调整、多种配色、音频引导等应用层定制——**不能仅依赖系统级无障碍设置** [A1-CTM, urban-mobility study]。
- 自闭症谱系异质性极高，"每个孩子需求独特"，单一方案不适用→需可适配 [IEEE 11168421]。

### 1.6 导航与认知负荷 Navigation & cognitive load
- 清晰的页面结构与导航（**NVG**）、最短关键路径、减少干扰 [A1-NVG/EGM]。
- 多发展水平的内容选项（满足不同能力层）[A5]。

### 1.7 触屏与交互 Touchscreen interaction
- 触屏对部分 ASD 儿童更易用；针对移动端的交互推荐（**ITST**）[A1-ITST, A15, IEEE 7462265]。

> **方法学趋势（ASD）**：从启发式（Khowaja & Salim 改良 Nielsen 15 条 [A3]）→ 指南（GAIA [A1]）→ **参与式/共同设计**（青少年 [A10]、成人 [A13]、适配工作坊 [A14]）。成人、职场 [A12]、自我干预 [A11] 是新增长点。

## 2. 其他人群的设计原则 — RQ2

### 2.1 智力/学习障碍 IDD
核心论断：**WCAG 是起点但不针对 IDD，合规不保证认知可用** [B1]。
- **专用设计模式**：简化导航、清晰标签、错误预防 [B1]。
- **能力本位设计 (ability-based design)**：以能力而非缺陷为中心 [B4]。
- **降低内在认知负荷**：减少任务步骤、优化标准设计模式 [B5]（Easy-to-Read 用户）。
- **创伤知情设计 (trauma-informed)**：与自我倡导组织合作、设计选择可能触发创伤、加入旁白、提供创伤感知的选择 [B2]。
- **自然用户界面**：触控/语音/无触可达性差异 [B7]；多步任务的独立完成支持 [B8]；可适配应用开发 [B9]。
- **利益相关者全程参与**（异质需求难以被单一设计满足）[B3]。

### 2.2 痴呆 / 老年认知衰退 Dementia & Aging
- **自适应 UI**：导航自适应 + 内容自适应，基于情境与用户画像 [C4]。
- **降低记忆依赖**：智能提醒、任务提示（离家提醒、个性化提醒）[C3, C5]——直接对应 COGA "processes do not rely on memory"。
- **应对多重感官衰退**：视/听/言语/灵巧度同时变化；支持可调节刺激强度、内置设置、设备切换 [C1]。
- **用户中心 + 照护者生态**：照护者作为设计对象，关注健康公平 [C2, C3]。

### 2.3 ADHD / 阅读障碍 Neurodivergent
- **可读性指南**：与设计/阅读障碍专家共建的 **61 条 Web 可读性指南** [D2]；阅读障碍 **41 条指南**（奠基 [D6]）。
- **成人 ADHD 包容性 Web 设计指南** [D1]。
- **注意力自适应/降低认知负荷**：行为注意检测可显著降低认知负荷、提升理解（Wizard-of-Oz）[D5 及相关]。
- **执行功能视角**：界面常"by design"制造执行功能障碍；AI 支持 vs 制度性障碍 [D5]。
- **共同设计 + 教育**：与神经多样用户共同设计 [D3]，并将无障碍纳入计算机教育 [D4]。

### 2.4 获得性脑损伤 / 失语 TBI & Aphasia
- **多种针对性辅助模式**：SMART-TBI 的写作辅助、解读辅助、过滤模式、**专注模式 (Focus Mode)**、定制 [E1]——专注与定制呼应 ASD/ADHD。
- **图像/多模态叙事**支持沟通 [E4-BG]；音频媒体可达性新设想 [E2]。
- **单被试/个案驱动**：让用户主导文本、指令等界面参数的设定（认知康复传统）[E3-BG]。

## 3. 综合：跨人群共性 vs 特异性 — RQ3

### 3.1 跨人群共性原则（以 COGA 八目标为骨架 [F1]）

| 共性原则 | 含义 | 证据人群 |
|---|---|---|
| **清晰可理解的内容** | 简单词汇、直接语态、字面语言、succinct | ASD[A4]·IDD[B5]·ADHD/读障[D2]·TBI[E1] |
| **降低记忆依赖** | 提醒、不靠计算/记忆、简化登录 | 痴呆[C5]·IDD[B8]·(COGA 通用) |
| **降低认知负荷 / 支持专注** | 减步骤、最短路径、限制干扰、Focus Mode | ASD[A1]·ADHD[D5]·TBI[E1]·IDD[B5] |
| **可预测 + 一致** | 一致视觉/交互、稳定不突变、清晰步骤 | ASD[A8]（最强）·IDD[B1]·痴呆[C4] |
| **防错 + 反馈** | 验证、可撤销、状态可见、明确反馈 | ASD[A1]·IDD[B1]·(COGA) |
| **帮助与支持** | 人工帮助、备用内容、导航协助 | 全人群 (COGA[F1]) |
| **可定制 / 个性化** | 字体/配色/音频/简化版、应用层而非仅系统级 | ASD[A1]（最强）·痴呆自适应[C4]·TBI[E1] |
| **多模态冗余表征** | 图文音并行、不绑定单一格式 | ASD[A1-RR]·失语[E4]·读障[D2] |
| **参与式/能力本位** | 与目标人群共同设计、以能力为中心 | ASD[A10,A13]·IDD[B3,B4]·神经多样[D3] |

### 3.2 人群特异性（最突出者）

| 人群 | 最具特异性的原则 | 与自闭症的关系 |
|---|---|---|
| **自闭症 ASD** | **感官调节** + **抗变化/可预测** + **字面化语言** + **深度定制** | — |
| **IDD** | **创伤知情** + **能力本位** + 极简化 | 共享：简化、参与式；差异：ASD 更强调感官 |
| **痴呆/老年** | **降低记忆依赖** + **多重感官衰退适配** + 照护者生态 | 共享：自适应/定制；差异：渐进性退化、记忆为核心 |
| **ADHD/读障** | **注意力自适应** + **可读性（排版/字体）** + 执行功能支持 | 与 ASD 高度重叠（专注、定制）；"neurodivergent"常合并研究 |
| **TBI/失语** | **多模态沟通替代** + 个案化参数 + 语言产出/理解辅助 | 共享：专注模式、冗余表征 |

**关键洞察**：
- 自闭症的"专注"与"定制"需求与 ADHD/TBI **高度共振**，"neurodivergent"研究常将三者合并 [D3, D4]。
- 自闭症独有且最强的是**感官调节**与**对界面变化的敏感**[A8, A11]——这是其他认知人群文献中较少作为首要原则出现的。
- **可定制/个性化是真正的"超级原则"**：几乎所有人群都需要，但内容不同（ASD=感官/配色；痴呆=自适应难度；ADHD=注意力适配；TBI=输入输出辅助）。

## 4. 方法学与证据空白 — RQ4

- **来源类型**：专家共识指南（GAIA[A1]、可读性 61 条[D2]、ADHD[D1]）；改良启发式（Nielsen→ASD[A3]）；参与式/共同设计（[A10,A13,A14,B3,B4,D3]）；实证评估（[A8,B7,C1,D5]）。
- **趋势**：2015→2025 由"为用户设计"向"与用户共同设计"迁移；近两年（2024–25）出现 **AI/LLM 介入认知可达性**（文本简化、执行功能 AI 支持）[D5, F5] 与 **WCAG 2.2 新增认知相关成功标准** [F4]。
- **空白**：(1) 原则多为单人群、小样本，缺**跨人群对照验证**；(2) 自闭症**成人/职场/真实情境**仍少 [A12]；(3) 指南**落地与合规度评估**工具仍不成熟 [F2]；(4) 文化/语言多样性（多为英语/欧美语境）；(5) **AI 时代**的新风险（界面频繁更新对 ASD 的冲击 [A8]）。

## 6. 一句话结论

> 认知无障碍存在一套以 **W3C COGA 八目标**为骨架的跨人群共性设计原则（清晰语言、降记忆、降负荷/专注、可预测、防错反馈、帮助、可定制、多模态、参与式）；**自闭症**在此之上以**感官调节、抗界面变化、字面化语言、深度定制**为最鲜明的特异性，且与 ADHD/TBI 在"专注+定制"上高度共振。未来缺口在跨人群验证、成人/职场情境与 AI 时代的新设计责任。
